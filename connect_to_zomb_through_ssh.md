# Подключение личного (корпоративного) ноутбука к «зомбу» по SSH через Ethernet

**Инструкция:** прямое соединение кабелем (point-to-point) без роутера. Пример команд приведён для Linux на «зомбе» и для macOS на ноутбуке, но логика одинакова для любой ОС.

---

## Что понадобится

- Доступ к консоли «зомба» (локально, через монитор-клавиатуру или уже открытая сессия).
- Ethernet-кабель.
- Если «зомб» — ноутбук с USB‑C: переходник USB‑C → Ethernet.
- SSH-доступ на «зомб» (логин/пароль известны).

---

## 1. Определить сетевой интерфейс на «зомбе»

1. На «зомбе» открой терминал и посмотри список интерфейсов до подключения кабеля:
   ```sh
   ip -br a
   ```
   Или для более подробного вывода:
   ```sh
   ip a
   ```
2. Подключи Ethernet‑кабель к «зомбу».
3. Снова выполни:
   ```sh
   ip -br a
   ```
4. Сравни вывод до и после, найди интерфейс, который изменился:
   - стал `UP` вместо `DOWN`, и/или
   - получил новый IP, и/или
   - появился новый интерфейс (часто при USB‑C адаптерах).

Запомни имя интерфейса (например: `eth0`, `enp0s31f6`, `ens33`, `enx...`).

---

## 2. Назначить статический IP на «зомбе»

Назначим «зомбу» IP в локальной подсети, например `192.168.120.0/24`.

Выполни на «зомбе» (подставь свой интерфейс):

```sh
sudo ip addr add 192.168.120.121/24 dev <INTERFACE>
sudo ip link set <INTERFACE> up
```

Пример:

```sh
sudo ip addr add 192.168.120.121/24 dev enx00e04c1523a8
sudo ip link set enx00e04c1523a8 up
```

### Проверка

```sh
ip -br a
```

Должно быть видно `192.168.120.121/24` на нужном интерфейсе.

> **Важно:** эта настройка временная (до перезагрузки); чтобы закрепить, используйте NetworkManager/netplan и т.п.

---

## 3. Назначить IP на ноутбуке (macOS пример)

1. Открой: **System Settings → Network → (твой Ethernet/USB 10/100/1000) → Details → TCP/IP**
2. Установи:
   - **Configure IPv4:** Manually
   - **IP Address:** `192.168.120.122` (или любой свободный адрес в подсети)
   - **Subnet Mask:** `255.255.255.0`
   - **Router:** можно оставить пустым (для прямого подключения не обязателен)
3. Сохрани изменения (Apply / OK).

> Главное: IP ноутбука должен быть в той же подсети, что и IP «зомба». Например, если у «зомба» `192.168.120.121/24`, то ноутбук — `192.168.120.xxx/24`.

---

## 4. Проверить связь (ping)

С ноутбука:

```sh
ping 192.168.120.121
```

Если пакеты идут — сеть поднята.

Если не идёт:

- проверь, что оба IP в одной подсети `/24`;
- проверь, что интерфейс на «зомбе» `UP`;
- проверь кабель/адаптер;
- иногда помогает переподключить кабель и повторить шаг 1.

---

## 5. Подключиться по SSH

С ноутбука:

```sh
ssh ymbot@192.168.120.121
```

- Логин: `ymbot`
- Пароль: тот же, что используешь для входа на «зомб» после рестарта.

---

## 6. Полезные заметки

### Про «порт»

Обычно SSH работает на порту 22, и его не нужно указывать. Если используется другой порт:

```sh
ssh -p <PORT> ymbot@192.168.120.121
```

### Повторное подключение в будущем

Если IP на «зомбе» назначался командой `ip addr add`, то после перезагрузки он пропадёт.
Чтобы сделать его постоянным, нужно закрепить в системе:

- Ubuntu Server: netplan
- Desktop Linux: NetworkManager (`nmcli`)
- Debian без NM: `/etc/network/interfaces` (редко в новых системах)

---

## Быстрый чек-лист

1. На «зомбе»: `ip -br a` → найти интерфейс после подключения кабеля
2. На «зомбе»: `sudo ip addr add 192.168.120.121/24 dev <iface>`
3. На ноутбуке: вручную `192.168.120.122 / 255.255.255.0`
4. `ping 192.168.120.121`
5. `ssh ymbot@192.168.120.121`
